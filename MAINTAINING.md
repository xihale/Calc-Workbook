# 维护说明

这份文档记录 `Calc-Workbook` 的日常维护约定，重点是避免 Typst 数学语法和答案宏再次出现排版回归。

## 快速检查

优先使用下面这条命令检查当前改动：

```bash
typst compile "main-answer.typ"
```

如需同时生成习题版，再运行：

```bash
typst compile "main-question.typ"
```

## 内容入口

- 新题目优先加到对应 `chapters/chapter-*.typ`
- 章节汇总由 `chapters/all.typ` 控制
- 不要直接在 `main-question.typ` 或 `main-answer.typ` 中写题目内容

## 模板宏约定

- `#question(...)`：题目容器，负责题目、答案和解析的布局
- `#ans[...]`：题干中的填空答案
- `#choice[...]`：选择题或判断题答案
- `template.typ` 中的答案显示逻辑已经区分“题干内答案”和“单独答案行”，修改前先理解现有状态判断

## Typst 数学语法约定

### 必须遵守

- 上下标分组使用 `^(...)` 和 `_(...)`
- 不要再写 LaTeX 风格的 `^{...}` 或 `_{...}`
- 需要分段函数时，使用合法的 `cases(...)` 结构，确保每一项都完整成对
- 句号、分号等中文标点尽量写在数学环境外侧

### 高风险写法

- `#ans[#h(...)]` 这类把占位空白塞进答案宏的写法
- `x y ^2`、`x y ^3` 这类上标前多出空格的写法
- 把 `quad` 当普通数学内容直接写进 `$...$`
- 结构不完整的 `cases(...)`，尤其是条件中带 `if`、比较符和分式时

## 内容编辑建议

- 一题内如果已经用 `#ans[...]` 给出填空答案，通常不再额外写同义的 `a:`
- 解析尽量保持一行一步，避免复制后重复一整行推导
- 长公式优先先写对，再调版式；不要为了凑视觉效果引入奇怪空格

## 辅助脚本

可以用下面的脚本检查数学环境中的非 ASCII 字符：

```bash
python "check_non_ascii.py"
```

## 产物管理

- 生成的 PDF 已由 `.gitignore` 忽略
- 提交前优先确认源文件 `.typ` 的结构和数学语法，而不是只看 PDF 是否“勉强能编译”
